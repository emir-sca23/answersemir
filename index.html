ЧТО ТАКОЕ GIT WORKFLOW?

это процесс работы вашей команды, он нужен для того, чтобы несколько людей из вашей команды параллельно работали над кодом,не мешая друг другу.
Также с ним легко отслеживать все изменения коммитов,веток и т.д. и чтобы он не разрушал основной код(main), и оставался стабильным.

GIT MERGE and GIT REBASE

git merge сохраняет всю историю коммитов и показывает точку слияния двух веток.
git rebase переписывает всю историю перенося изменения одной ветки на вершину другой ветки,где не будет история коммитов

GIT FETCH and GIT PULL

git fetch - загружает изменения из удаленного репозитория без их применения к ветке.
git pull - загружает и автоматически сливает изменения с веткой. git fetch подходит для проверки изменений, git pull подходит если хочешь быстро обновить свой код.
